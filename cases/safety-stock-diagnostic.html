Safety-Stock Diagnostic

New SKUs disappeared from reorder review because several weak control layers interacted: formula behavior, extraction logic, classification logic, filters, workbook governance, and active macro version control.

What this proves

Root-cause diagnosis across data, workbook, and automation layers.

This case demonstrates process-improvement depth: diagnosing why a system produces a bad outcome, then redesigning controls so the issue cannot silently recur.

The visible problem was a replenishment issue. The real issue was control failure.

New SKUs with no usage history were treated as inactive or hidden. The system appeared stable because multiple silent failures reinforced each other.

FormulaNo usage history can collapse calculated safety stock.Near-zero output creates false confidence.
MacroZero usage was classified as DEAD.No explicit no-history state existed.
WorkbookActive logic was not always edited logic.Workbook-embedded code kept old behavior alive.

Fix pattern

LogicRebuild fragile logic in Power Query.Less vulnerable to overwritten worksheet formulas.
ClassificationAdd NO-USAGE state.New items require exception review, not dead-demand routing.
ResilienceUse dynamic header lookup.Avoid silent filter failure from column shifts.
DebuggingAdd diagnostic columns.Make intermediate logic visible during troubleshooting.
