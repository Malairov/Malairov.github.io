Executive summary

The issue was not one bad formula. It was a control-system failure.

New SKUs with weak or no usage history were vulnerable to being treated as inactive or hidden. The operating risk came from several layers reinforcing the wrong outcome.

Operating context

Tool layerExcel, Power Query, VBA.Inherited workbook logic carried hidden operating risk.
Decision layerSafety-stock and review logic.The tool influenced replenishment planning decisions.
Risk layerSilent exclusion of new items.Items could disappear before anyone reviewed them.

Problem and constraints

The visible issue looked like a replenishment problem. The actual problem was that formulas, classification logic, filters, overwritten cells, and active macro versions could interact and hide valid exceptions.

ConstraintNo usage history is not dead demand.New items needed exception handling.
ConstraintWorksheet formulas can be overwritten.Corrected logic may not run if cells are static values.
ConstraintHardcoded filters are brittle.Column shifts can silently hide the wrong items.
ConstraintMacro version control matters.The edited code is not always the active code.
